International days

Description

This project is the next step of a project for the course Requirements Analysis. In this subject the goal was to analyse the must haves of a application of a customer. Another group had to make the web application. My group got a analysis for the International days.

Goal

Create a web application with a group of students using Laravel, Phpmyadmin and Github. While we are working on this project we will learn to work in team, use our knowledge of Laravel and Github.

Example of one of my contributions

Manage activities

One of my contributions is the functionality "Manage activities". This page shows a table with all the active activities. You can add, edit and delete an activity. Each activity has a activity type. The activity types managing works the same as the activities. The input fields have client side and server side input checking. The user will also get a specific error message if a field is not filled in or is filled in wrong.

Conclusions

This is one of the first big projects I had to make. Thats why this was a big step in working as a team. Using branches in github, dividing tasks with JIRA and communicating issues with eachother. While we were developing the application we had 2 meetings with the customer. This was very usefull to get continiues feedback. One big positive thing for me was that I learned a lot of the team members.
