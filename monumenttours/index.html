Monument tours

Description

This group project is made for the course Advanced Programming Topics. In this course we learned to create Docker images application, using tools like SonarCloud and Selenium and the power of Github actions. We were allowed to choose the theme ourselves. We chose to make something about tours of all kinds of monuments around the world. As a extra we made a simple webapplication in Angular to show our microservice API.

Goal

Create in a group of 2 students a microservices project. We had to atleast have 2 microservices and a edge service. The application needs to use MongoDB and another non-MySQL Database. Each microservices had minimum number of GET, POST, PUT and DELETE calls. We also had to foresee unit and integration tests. At last we had to implement a CI/CD deployment that runs the tests, upload a .JAR as a artifact and send a dockerimage to Docker hub and use SonarCloud.

My contributions

Docker and CI/CD

I Wrote a dockerfile for each microservices we made and the pipeline that gets executed by Github actions. The dockerfile pulled a image and made it ready for our application. The CI/CD file was set to run on push, to make use of ubuntu. It also told the image what it needs to use and install, what the sonarcloud token are (using Github secrets) and finally push the image to Docker hub.

Writing part of the api

This part of the project was simply making a webapi using Springboot. Because we choose to make 3 microservices I helped writing a bit of the api enpoints. The code exsists out of a controller containing all the endpoints, a model that defines the properties of the subject of the microservice and a repositoy to do calls to de database image.

Kubernetes

The usefull part of making microservices is that if one part of the API gets a lot of request, you can simply start another image with Kubernetes and the traffic gets devided between the images. This is setup by a directory named k8s. This direcotry contains all the config files for each microservice.

Conclusions

Making Docker images and microservices is really usefull. That way you can scale your application very easily. With Docker you can run your application on all the machines without the need of installing all kinds of packages. At first this course scared me because of the words Docker and Kubernetes but once you understand how everythig works it is not that hard anymore. In this project I got the chance to increase my experience with Docker, Kubernetes and I learned that Github is not just a version control tool.
